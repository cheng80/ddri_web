# DDRI 웹서비스 화면 설계서

작성일: 2026-03-16  
목적: Flutter Web 기준으로 사용자 페이지, 관리자 페이지, 통계 페이지의 화면 구조와 주요 컴포넌트를 정의한다.

## 1. 공통 원칙

### 범위 밖 (절대 포함 금지, 강력 제약)

> DDRI는 따릉이 전용 대여 앱이 아니다. 따릉이 사용자가 **비로그인 상태로 조회만** 하는 정보 앱이다.

- **이용권** (구매, 조회, 결제, 구독) — **절대 금지**
- **사용자 my** (회원가입, 로그인, 프로필, 즐겨찾기, 마이페이지) — **절대 금지**

→ 대여소 조회·관리·통계만 제공하는 **비로그인 공개 웹**이다.

---

- 링크 기반 진입
- `GetRoute` 기반 페이지 분리
- 데스크톱 우선 웹 화면 설계
- 동일한 상단 내비게이션 유지
- 데이터 로딩, 빈 상태, 에러 상태를 각 페이지에 명시

공통 상단 메뉴:

- `사용자`
- `관리자`
- `통계`

## 3. 사용자 페이지 설계

경로: `/user`

### 화면 목적

> 사용자가 **내 근처** 또는 **지정 주소 근처**에서, **지정 시간대**에 자전거가 얼마나 있는지 **예측**해서 보여주는 것. 가능하면 **길찾기** 제공.

상세 설계: [08_ddri_user_page_spec_detail.md](08_ddri_user_page_spec_detail.md)

### 주요 섹션

1. 검색/입력 영역 (3가지 필수)
   - **내 위치 찾기** (Geolocation API → 위경도)
   - **주소 찾기** (kpostal_plus 또는 카카오 우편번호)
   - **시간대 선택** (날짜·시간, 예측 기준 시각)
2. 가까운 대여소 목록 (거리순, 지정 시간대 기준 예측)
   - 대여소명
   - 거리 (m/km)
   - 현재 자전거 수
   - 예상 잔여 자전거 수
   - 대여 가능 여부 배지
   - **길찾기** 버튼 (가능하면)
3. 보조 정보 영역
   - 운영 상태
   - 예외 스테이션 안내 문구
   - 기준 시각 안내

### 상태 정의

- 초기 상태:
  - 검색 안내 문구 표시
- 조회 성공:
  - 결과 카드 표시
- 실시간 비노출:
  - `실시간 정보 없음` 메시지 표시
- API 실패:
  - 재시도 버튼 표시

## 4. 관리자 페이지 설계

경로: `/admin`

### 화면 목적

- 재고 부족 가능성이 높은 대여소를 한 화면에서 파악

### 주요 섹션

1. 상단 제어 영역
   - 기준 날짜/시간 선택
   - 군집 필터
   - 운영 상태 필터
   - 정렬 기준 선택
2. 요약 카드 영역
   - 전체 대상 대여소 수
   - 위험 대여소 수
   - 예외 스테이션 수
   - 평균 위험 점수
3. 메인 표 영역
   - 대여소명
   - 군집
   - 현재 재고
   - 예측 수요
   - 재고 차이값
   - 위험 점수
   - 재배치 우선순위
   - 운영 상태
4. 예외 스테이션 영역
   - 실시간 비노출 또는 비활성 후보 별도 표시

### 상태 정의

- 초기 상태:
  - 현재 시각 기준 기본 목록 로드
- 데이터 없음:
  - 조건에 맞는 대여소 없음 표시
- API 실패:
  - 표 대신 오류 메시지와 재시도 버튼 표시

## 5. 통계 페이지 설계

경로: `/stats`

### 화면 목적

- 서비스 전체 추이와 군집별 특성을 빠르게 확인

### 주요 섹션

1. 필터 영역
   - 날짜 범위 선택
   - 군집 선택
   - 지표 선택
2. KPI 카드 영역
   - 평균 예측 수요
   - 평균 잔여 자전거 수
   - 부족 위험 대여소 비율
   - 예외 스테이션 수
3. 차트 영역
   - 시간대별 수요 추이
   - 군집별 평균 수요 비교
   - 대여소별 위험도 상위 목록
4. 표 영역
   - 차트 원본 요약 테이블

### 상태 정의

- 초기 상태:
  - 최근 기준 기간 기본 차트 로드
- 로딩 상태:
  - 스켈레톤 카드/차트 표시
- 에러 상태:
  - 차트 대신 오류 메시지 표시

## 6. 공통 컴포넌트 목록

- 상단 네비게이션 바
- 검색 입력 컴포넌트
- 날짜/시간 선택 컴포넌트
- 상태 배지
- KPI 카드
- 데이터 테이블
- 빈 상태 뷰
- 에러 상태 뷰
- 로딩 스켈레톤

## 7. 반응형 기준

- 데스크톱:
  - 3열 또는 2열 레이아웃 사용
- 태블릿:
  - 카드 세로 배치 확대
- 모바일:
  - 표보다 카드형 목록 우선

1차 버전은 웹 중심이므로 데스크톱 레이아웃 완성도를 우선한다.

## 8. 구현 우선순위

1. 사용자 페이지
2. 관리자 페이지
3. 통계 페이지

이유:

- 사용자 페이지와 관리자 페이지가 핵심 기능이다
- 통계 페이지는 읽기 전용 시각화라 목업 데이터로도 먼저 구현 가능하다
