# DDRI 웹서비스 화면 설계서

작성일: 2026-03-16  
최종 갱신: 2026-03-19  
목적: Flutter Web 기준으로 사용자 페이지와 관리자 페이지의 화면 구조, 공통 원칙, 현재 구현 범위를 최신 상태로 정리한다.

## 1. 공통 원칙

### 범위 밖 (절대 포함 금지)

> DDRI는 따릉이 전용 대여 앱이 아니다. 따릉이 사용자가 비로그인 상태로 조회만 하는 정보 앱이다.

- 이용권:
  - 구매, 조회, 결제, 구독, 이용권 관리 금지
- 사용자 my:
  - 회원가입, 로그인, 프로필, 즐겨찾기, 마이페이지 금지

### 화면 구성 원칙

- 링크 기반 진입
- `GetX` 기반 페이지 분리
- 사용자/관리자 공통 상단 내비게이션 유지
- 모바일/태블릿/데스크탑 반응형 지원
- 로딩, 빈 상태, 오류 상태를 각 페이지에서 구분
- 색상/타이포/컴포넌트는 공통 Design Token 사용

공통 상단 메뉴:

- `사용자`
- `관리자`

## 2. 라우트 구조

- `/`
- `/user`
- `/admin`

설명:

- `/` 는 사용자 화면과 동일 진입점으로 동작
- 현재 `통계` 라우트는 없음

## 3. 사용자 페이지 설계

경로: `/user`

### 화면 목적

사용자가 내 위치 또는 지정 주소 근처에서, 지정 시간대 기준으로 대여 가능성을 확인하도록 돕는다.

상세 설계:

- `08_ddri_user_page_spec_detail.md`
- `10_ddri_user_page_map_ux_spec.md`

### 주요 섹션

1. 검색/입력 영역
   - 내 위치 찾기
   - 주소 검색
   - 날짜/시간 선택
   - 반경 선택
2. 지도/결과 구조
   - 단일 지도
   - 사용자 위치 중심
   - 반경 표시
   - 대여소 마커
3. 대여소 목록 영역
   - 거리순 정렬
   - 대여소명
   - 거리
   - 현재 자전거 수
   - 예측 대여량
   - 예상 잔여 수
   - 대여 가능 여부 배지
   - 길찾기 버튼
4. 보조 정보
   - 기준 시각 안내
   - 예외 스테이션 안내
   - 운영 상태 표시

### 상태 정의

- 초기 상태:
  - 위치 선택 안내 문구 표시
- 조회 중:
  - 로딩 인디케이터 또는 선형 진행 표시
- 조회 성공:
  - 대여소 카드 목록 표시
- 결과 없음:
  - 반경 조정 또는 위치 변경 안내
- 예외 발생:
  - 실시간 비노출 / API 오류 메시지 표시

### 현재 구현 메모

- 사용자 리스트는 카드형 목록으로 구현
- 위치 미선택/결과 없음/로딩 상태 UI가 반영되어 있음
- 목업 API 응답 기준으로 목록을 렌더링 중

## 4. 관리자 페이지 설계

경로: `/admin`

### 화면 목적

재고 부족 가능성이 높은 대여소를 시간대별로 파악하고, 재배치 우선순위 판단을 지원한다.

상세 설계:

- `13_ddri_admin_page_plan.md`
- `admin_layout_behavior.md`

### 주요 섹션

1. 상단 제어 영역
   - 기준 날짜 선택
   - 기준 시간 선택
   - 긴급 필터
   - 행정동 필터
   - 정렬 기준 선택
2. 요약 카드 영역
   - 전체 대상 수
   - 위험 대여소 수
   - 예외 스테이션 수
   - 평균 위험 점수
3. 메인 목록 영역
   - 대여소명
   - 행정동
   - 현재 재고
   - 예측 수요
   - 재고 차이값
   - 위험 점수
   - 재배치 우선순위
4. 예외 스테이션 영역
   - 실시간 비노출 등 예외 목록
5. 지도 영역
   - 현재는 플레이스홀더 수준
   - 추후 위험 대여소 시각화 확장 가능

### 상태 정의

- 초기 상태:
  - 현재 시각 기준 기본 목록 로드
- 조회 중:
  - 로딩 인디케이터 표시
- 결과 없음:
  - 조건에 맞는 대여소 없음 표시
- 예외 발생:
  - 목록 대신 오류 메시지와 재시도 유도

### 현재 구현 메모

- 관리자 목록은 테이블형 카드로 구현
- 항목 수가 적을 때는 shrinkWrap, 많을 때는 최대 높이 내부 스크롤 적용
- 예외 스테이션 영역과 지도 플레이스홀더가 목록 하단에 배치
- 군집은 현재 핵심 제어 항목보다 보조 정보로 후순위 처리

## 5. 공통 컴포넌트 목록

- 상단 네비게이션 바
- 공통 앱 스캐폴드
- 날짜/시간 선택 컴포넌트
- 상태 배지
- 요약 카드
- 데이터 목록/테이블
- 빈 상태 뷰
- 로딩 상태 뷰

## 6. 반응형 기준

- 모바일:
  - 세로 스택 중심
  - 표보다 카드형 목록 우선
- 태블릿:
  - flex-wrap 기반 배치
  - 리스트/표 혼합 가능
- 데스크탑:
  - 넓은 표와 2분할 레이아웃 우선

세부 기준은 `12_ddri_responsive_breakpoints_and_layouts.md`를 따른다.

## 7. 현재 구현 범위

- 공통 테마와 레이아웃 반영
- `/user`, `/admin` 라우트 구성
- 사용자 검색/입력 UI 및 리스트 UI 구현
- 관리자 제어 영역, 요약 카드, 목록, 예외 영역 구현
- FastAPI 목업 API 연동

미구현 또는 후속 작업:

- 실제 백엔드 데이터 연동
- 관리자 지도 고도화
- 통계 페이지
- E2E 검증

## 8. 구현 우선순위

1. 실시간 재고 연동
2. 예측 결과 연동
3. DB 멱등성 정리
4. 검증 및 시나리오 테스트
