# DDRI 관리자 페이지 기능 플랜

작성일: 2026-03-17  
목적: 관리자 페이지에 넣을 기능을 우선순위별로 정리하고, 개발 단계를 설계한다.

---

## 1. 목적과 대상 사용자

### 1.1 화면 목적

> **재배치 관리자가 이 페이지를 이용하는 이유**

- **시간대별 현황 파악**: 특정 시각 기준 전체 대여소 상태 확인
- **긴급 요하는 곳**을 빠르게 필터링·정렬해 재배치 판단 지원
- **예외 스테이션**(실시간 비노출·비활성)을 일반 대여소와 구분해 확인

> **참고**: 실제 재배치는 22~24시경 차량으로 이루어지나, 본 서비스는 **보여주기·판단 지원용**이다. 시간대 선택은 임의 시각 기준으로 동작한다.

### 1.2 대상 사용자

| 구분 | 설명 |
|------|------|
| **역할** | 강남구 따릉이 재배치·운영 담당자 |
| **핵심 질문** | "어느 스테이션이 언제 부족해질 가능성이 큰가?" |
| **행동** | 시간대별 현황 확인 → 긴급 구간 필터 → 재배치 판단 → (실제 재배치는 외부 프로세스) |

### 1.3 지역 구분 원칙

| 구분 | 용도 | 설명 |
|------|------|------|
| **행정동** | 운영·재배치 단위 | 담당 구역, 회의·보고 단위. 관리자가 동 단위로 움직인다 |
| **지역 특성** | ML 기반 그룹 | 이용 패턴·수요 특성이 비슷한 스테이션 묶음 (cluster00~04). 지도상 분포가 행정동과 완전히 일치하지 않음 |

→ 행정동 필터를 **주**로 두고, 지역 특성(군집)은 **보조 참조**로 활용. 표에는 둘 다 노출 가능.

### 1.4 사용자 페이지와의 차이

| 구분 | 사용자 페이지 | 관리자 페이지 |
|------|---------------|---------------|
| **초점** | 내 근처 / 지정 주소 근처 | 전체 161개 대여소 |
| **정렬** | 거리순 | 위험도·재배치 우선순위순 |
| **입력** | 위치 + 시간 | 기준 날짜/시간 + 필터 |
| **출력** | 예상 잔여 수, 대여 가능 여부 | 재고 차이, 위험 점수, 우선순위 |

## 2. 기능 목록 (우선순위별)

### 2.1 주가 기능 (MVP)

> **시간대별 현황 파악**과 **긴급 요하는 곳 필터링**이 핵심이다.

| 순위 | 기능 | 설명 | 데이터 소스 |
|------|------|------|-------------|
| 1 | **기준 날짜/시간 선택** | 시간대별 현황 파악용. 예측·재고 기준 시각 지정 | `base_datetime` |
| 2 | **긴급 필터** | 위험 대여소만 보기, stock_gap 음수만 보기 등 | 클라이언트/서버 |
| 3 | **대여소 목록 표** | 161개 전체 또는 필터 결과 | `station_risk_snapshots` |
| 4 | **핵심 컬럼 표시** | 현재 재고, 예측 수요, 재고 차이, 위험 점수, 재배치 우선순위 | API 응답 |
| 5 | **정렬** | 위험 점수·재배치 우선순위·재고 차이 등 기준 정렬 | 클라이언트/서버 |
| 6 | **예외 스테이션 구분** | 실시간 비노출·비활성 후보 별도 표시 또는 배지 | `operational_status` |
| 7 | **요약 카드** | 전체 수, 위험 대여소 수, 예외 수 | 집계 |

### 2.2 2차 (개발 여유 시)

| 순위 | 기능 | 설명 |
|------|------|------|
| 8 | **행정동 필터** | 동 단위 필터 (운영·재배치 단위) |
| 9 | **지역 특성(군집) 필터** | cluster00~04. ML 결과이지만 지역별 이용 패턴 반영 |
| 10 | **운영 상태 필터** | operational / 비노출 / 비활성 등 |
| 11 | **검색** | 대여소명·주소 검색 |
| 12 | **내보내기** | CSV/엑셀 다운로드 (재배치 회의용) |

### 2.3 3차 (추후 검토)

| 순위 | 기능 | 설명 |
|------|------|------|
| 13 | **지도 뷰** | 위험 대여소 지도 표시 (군집 색상 등 지역 특성 시각화) |
| 14 | **시간대 비교** | 여러 시간대 동시 비교 |
| 15 | **관리자 메모** | 대여소별 메모 저장 (DB 확장 필요) |

## 3. UI 구성 (MVP 기준)

### 3.1 레이아웃

```
┌─────────────────────────────────────────────────────────────────┐
│  [DDRI 로고]     사용자  관리자                                   │
├─────────────────────────────────────────────────────────────────┤
│  관리자 – 재배치 판단 지원                                        │
├─────────────────────────────────────────────────────────────────┤
│  [기준 날짜] [기준 시간]  [긴급만▼] [정렬: 위험점수▼]  [행정동▼]  │  ← 제어 영역
├─────────────────────────────────────────────────────────────────┤
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐            │  ← 요약 카드
│  │ 전체 161 │ │ 위험 23  │ │ 예외 3   │ │ 평균 0.4 │            │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘            │
├─────────────────────────────────────────────────────────────────┤
│  대여소명        │ 동     │ 지역특성│ 재고 │ 예측수요 │ 차이  │ 위험 │ 우선순위 │  ← 메인 표
│  ─────────────────────────────────────────────────────────────  │
│  역삼지하보도... │ 역삼동 │ c00  │  7   │   12    │  -5   │ 0.8  │   1     │
│  강남역 2번...   │ 역삼동 │ c01  │  3   │    8    │  -5   │ 0.7  │   2     │
│  ...            │       │      │         │       │      │         │
├─────────────────────────────────────────────────────────────────┤
│  [예외 스테이션 접기/펼치기]                                      │  ← 예외 영역
│  station_id 2314, 2323, 3628 – 실시간 비노출                      │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 표 컬럼 정의 (MVP)

| 컬럼 | 영문 필드 | 설명 |
|------|-----------|------|
| 대여소명 | station_name | 스테이션 이름 |
| 행정동 | district_name | 운영·재배치 단위 |
| 지역특성 | cluster_code | cluster00~04. 이용 패턴 기반 그룹 (참조용) |
| 현재 재고 | current_bike_stock | 실시간 보유 대수 |
| 예측 수요 | predicted_demand | 해당 시간대 예측 대여량 |
| 재고 차이 | stock_gap | 재고 − 예측수요 (음수면 부족) |
| 위험 점수 | risk_score | 0~1, 높을수록 위험 |
| 재배치 우선순위 | reallocation_priority | 1, 2, 3... (1이 최우선) |
| 운영 상태 | operational_status | operational / 비노출 / 비활성 |

### 3.3 요약 카드 (MVP)

| 카드 | 설명 |
|------|------|
| 전체 대여소 수 | 필터 적용 후 대상 수 |
| 위험 대여소 수 | 위험 점수 임계값 초과 또는 stock_gap < 0 등 |
| 예외 스테이션 수 | 실시간 비노출·비활성 |
| 평균 위험 점수 | (선택) 전체 또는 필터 대상 평균 |

## 4. 데이터·API 요구사항

### 4.1 관리자 목록 API (예상)

**요청**

- `base_datetime`: 기준 시각 (ISO 8601)
- `urgent_only`: (선택) 긴급(위험) 대여소만
- `district_name`: (선택) 행정동 필터
- `cluster_code`: (선택) 지역 특성(군집) 필터
- `sort_by`: 위험점수 | 재배치우선순위 | 재고차이
- `sort_order`: asc | desc

**응답** (기존 01_ddri_flutter_web_service_preparation.md 기준)

```json
{
  "base_datetime": "2026-03-16T18:00:00+09:00",
  "summary": {
    "total_count": 161,
    "risk_count": 23,
    "exception_count": 3,
    "avg_risk_score": 0.42
  },
  "items": [
    {
      "station_id": 2328,
      "station_name": "르네상스 호텔 사거리 역삼지하보도 7번출구 앞",
      "district_name": "역삼동",
      "cluster_code": "cluster00",
      "current_bike_stock": 7,
      "predicted_demand": 12.0,
      "stock_gap": -5.0,
      "risk_score": 0.72,
      "reallocation_priority": 1,
      "operational_status": "operational"
    }
  ],
  "exceptions": [
    { "station_id": 2314, "reason": "실시간 비노출" },
    { "station_id": 2323, "reason": "실시간 비노출" },
    { "station_id": 3628, "reason": "실시간 비노출" }
  ]
}
```

### 4.2 데이터 소스

| 항목 | 경로 |
|------|------|
| 실시간 재고 | OA-15493 bikeList → `realtime_station_stock` |
| 예측 수요 | `station_demand_forecasts` 또는 실시간 계산 |
| 재배치 지표 | `station_risk_snapshots` 또는 조합 계산 |
| 예외 스테이션 | `station_api_mappings` + 운영 상태 lookup |

## 6. 상세 기능은 개발 중 확정

- 위험 점수·재배치 우선순위 **계산식**은 ML/후처리 규칙 확정 후 반영
- **임계값**(예: risk_score > 0.5를 위험으로 볼지)은 개발 중 결정
- **표 컬럼** 순서·표시 형식은 실제 UI 구현 시 조정
- **반응형**: 데스크탑 우선, 태블릿/모바일은 표보다 카드형 목록 고려
- **행정동**: stations 테이블에 district_name 또는 행정동 코드 필요. 없으면 주소 파싱 또는 별도 매핑
